# JANUS フェーズ1 MVP 実装メモ

対象要件: 「旧世界統合作戦計画支援システム JANUS 要件定義書」版0.1（2026-07-19）

## 実装範囲

| 機能 | 実装内容 |
|---|---|
| F-01 | 1536×1024旧世界地図上の作戦状況図。目標、経路、区域、フェーズラインを保存し、50%〜800%のズームとパンに対応 |
| F-02 | 作戦名、期間、主体、敵、指揮官意図、最終状態、目的、失敗条件、ROE、制約等の構造化入力 |
| F-03 | ORBAT部隊のID参照割当、任務区分、配下人員・兵器ロールアップ |
| F-04 | ローカルまたは外部LLMによる直接型・形成型・間接型の3 COA、採用案選択、構想本文修正 |
| F-06 | 必須項目、期間、距離÷速度、利用可能時間、部隊・兵器参照、重複・親子割当、作戦図・COAの検査 |
| F-08 | 五段落命令、付属書、JANUS JSON、印刷/PDF、提出ロック |
| F-09 | 2計画の並列表示、期間・地域・同一部隊の衝突候補表示 |
| F-10 | 提出時のORBAT・カタログ・定義・計画書本文の固定保存 |

## 意図的な制約

- 外部LLMはOpenAI互換の `/models` と `/chat/completions` に限定する。LM Studioを既定プリセットとし、外部ホストにはHTTPSを要求する。
- 外部LLM呼出は接続テスト、COA生成、レッドチーム検証の明示操作時だけ行う。既定のローカル生成は外部送信を行わない。
- LLMには現在の作戦、割当部隊、その集計装備、作戦図要約だけを送り、他作戦、審判計画、無関係なカタログ品目は送らない。
- LLM出力は助言であり、許可ID検証後に取り込む。距離・時間・参照整合性と提出可否は決定論的ルールエンジンが判定する。
- APIキーはページ内メモリだけに保持し、保存・出力・AIログ記録を行わない。
- 地図尺度は小マス100km、太線10マス1,000kmである。現段階では平面直交座標として距離を計算し、球面・測地線補正は行わない。
- 兵器カタログに速度、航続距離、射程、輸送力、弾薬、燃料の構造化列がないため、兵器個別の到達性・兵站検査は警告として保留する。
- 三層情報モデル、認証、9軸評価、タイムライン衝突抽出、裁定結果のORBAT反映はフェーズ2以降の対象である。

## データ保持

作戦はプロジェクトJSONの `operations` 配列へ保存する。外部LLMの検証結果は `redTeam`、生成履歴は `aiLogs` に保存するが、APIキーとプロンプトは保存しない。提出時は `snapshot` に参照部隊・兵器・定義を保存し、`frozenOrder` に五段落命令本文を固定する。提出済み計画を編集したい場合は「複製」で下書きを作成する。

地図図形は `mapVersion: 2` と画像座標で保存する。v0.6.0で作成した `mapVersion` のない図形は読込時に自動変換する。経路距離は `100 / 4.577` km/pxとして算出し、複数経路では最長値を計画移動距離に採用する。表示倍率は50%〜800%で、スライダー、ボタン、Ctrl/⌘＋ホイールから変更できる。Space＋ドラッグまたは中ボタンドラッグで表示領域を移動する。ズームとパンは表示状態であり、保存座標や距離計算を変更しない。
