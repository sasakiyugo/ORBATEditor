# 要件実装状況 v0.1

基準文書: 「ORBATエディタ 要件定義書」版0.1（2026-07-18）

## 実装済み

| 要件群 | 対応内容 |
|---|---|
| FR-101〜106 | 複数ルートのツリー、追加・削除・改名・並べ替え・隷属変更・配下複製・属性編集・検索・折りたたみ |
| FR-201〜203 | 既定12階梯、旅団以上属性、ユーザー定義階梯の追加・編集 |
| FR-301〜305 | カタログCRUD、編集可能区分、CSV取込、配備先集計、参照中削除の拒否 |
| FR-401〜404 | 旅団未満のカタログ割当・数量編集・人員入力、旅団以上で直接割当UIを非表示 |
| FR-501〜505 | 全子孫の品目別・人員集計、区分小計、直接値と総計の併記、即時再計算 |
| FR-601〜605 | APP-6D準拠の既定兵科グリフ、階梯記号、縦横比を固定した4所属フレーム、部隊名・集計アンプリファイア |
| FR-701〜706 | 上下型部分組織図、深度制限、情報切替、ズーム・パン、SVG / PNG出力 |
| FR-801〜804 | 単一JSON保存読込、集計CSV / Markdown、離脱警告、Undo / Redo |
| FR-901〜903 | 循環する親候補の除外、階梯逆転警告、不正直接割当の検出 |

## 次版候補

- FR-606: ユーザー定義SVGグリフの登録
- ドラッグ＆ドロップによる並べ替え・隷属変更（現版はボタンと上級部隊選択で同じ操作を提供）
- 組織図での枝単位の一時折りたたみ（現版は起点・深度で表示範囲を制御）

## 受け入れ試験

`npm test` は次を自動検証する。

1. サンプル旅団の人員が1,700名である。
2. 主力戦車88両、HIFV-40 40両が集計される。
3. 第1戦車大隊を44両から31両に変更すると、旅団合計が75両になる。
4. 旅団では直接割当UIが非表示である。
5. 組織図に起点以下の部隊ノードが描画される。
6. 画面実行中にエラーが発生しない。
