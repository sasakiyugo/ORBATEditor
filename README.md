# ORBAT Editor

Current version: **0.8.0**

部隊階層、装備カタログ、自動集計、NATO APP-6風の兵科記号付き組織図を一元管理する、ローカル完結型ORBATエディタです。

## 使い方

1. リポジトリをダウンロードまたはZIPで取得します。
2. `index.html` を Chrome / Edge で開きます。
3. 編集後、ヘッダーの「JSON保存」でプロジェクトを保存します。

以前のZIPを展開済みの場合は、フォルダへの上書きではなく旧フォルダを削除してから最新版を展開してください。アプリUIは常時ダーク、組織図キャンバスと画像出力は白背景です。

依存関係、ビルド、サーバ、アカウントは不要です。既定のローカル生成では外部送信を行いません。外部LLMはユーザーが接続設定を保存し、作戦画面で明示的に実行した場合だけ呼び出します。

## 初版の主な機能

- 複数ルート対応の部隊ツリー（追加、削除、改名、並べ替え、隷属変更、配下ごと複製、検索、折りたたみ）
- 編集可能なエシュロン・兵科定義
- 兵器カタログのCRUD、CSVインポート、使用箇所表示、参照中削除の防止
- `兵器価格一覧(1).csv`から価格を除外した兵器1,651品目（名称・原種別・生産国）を初期収録
- 137の細分類を16の大分類へ整理し、元の分類は「原種別」として保持（収録兵器の「その他」は0件）
- 大規模カタログ向けの名称・区分・生産国検索と300件単位表示
- 旅団未満への装備・人員の直接割当
- 兵器割当時の区分フィルター（区分別の登録件数を表示）
- 旅団以上の全子孫ロールアップと区分別小計
- 所属フレーム、兵科、エシュロンを組み合わせたSVG兵科記号
- APP-6D準拠の所属フレーム形状と縦横比（味方3:2、敵・中立・不明は正方形境界）
- 常時ダークモード（ライトモードは廃止）
- 部分組織図、表示深度、人員・主要装備表示、ズーム・パン、エシュロン記号と重ならない接続線
- 組織図のSVG / PNG出力
- 単一JSONの保存・読込、集計CSV / Markdown出力、Undo / Redo、未保存離脱警告
- 循環参照の防止、階梯逆転と不正な直接割当の警告
- JANUS作戦タブ：構造化された作戦下書き、ORBAT部隊割当、旧世界地図上への作戦目標・経路・区域・フェーズラインの描画
- 直接型・形成型・間接型の3 COA生成、距離・時間・参照整合性の決定論的検査
- 五段落命令の生成、JANUS JSON出力、ブラウザ印刷によるPDF保存、提出ロックと情報スナップショット
- 同一プロジェクト内の2計画を並べる審判比較（勝敗は自動判定しない）
- LM StudioまたはOpenAI互換APIによる外部LLM接続、3 COA生成、採用COAのレッドチーム検証
- 外部LLMへ送信する情報を現在の作戦・割当部隊・参照装備へ限定し、応答中の未提供IDを拒否

初期プロジェクトには受け入れ試験用の師団・旅団・3個大隊が入っています。旅団の初期集計は、主力戦車88両、HIFV-40 40両、人員1,700名です。

初期カタログは旧世界固有兵器HIFV-40を含めて全1,652件です。固有兵器を指さない旧サンプル「主力戦車」「小銃」は削除し、サンプル戦車大隊にはM1A2を割り当てています。旧版JSONを読み込んだ場合は、旧「主力戦車」の数量をM1A2へ引き継いで移行します。価格情報は保存しません。

## JANUS 作戦計画支援

「作戦」タブでは、作戦基本情報、参加部隊、任務区分、作戦図形を入力し、3案のCOAと五段落命令を作成できます。計画の数値と装備はORBAT・兵器カタログのIDを参照し、提出時に参照データと計画書本文を固定します。後から編成を変更しても、提出済み計画は変化しません。

作戦状況図には旧世界地図を内蔵している。地図の小マス1個を100km、太線間隔の10マスを1,000kmとして扱う。描画点は小マスへスナップでき、進出経路の折線距離を自動計算して計画移動距離・所要時間検査・五段落命令へ反映する。50%〜800%のズームに対応する。

COA生成は、完全オフラインの構造化テンプレートと外部LLMを選択できます。外部LLMの応答も、未割当部隊IDや未提供兵器IDを参照していないか検証してから取り込みます。距離・時間・参照整合性の計算は従来どおりローカルの決定論的ルールエンジンが担当し、LLMの数値を検査結果として信用しません。兵器個別の航続距離・射程検査は、カタログに性能属性を追加する次段階の対象です。

## 外部LLM接続

1. 「設定」→「外部LLM接続」を開きます。
2. LM Studioの場合はローカルサーバを起動し、モデルIDを入力します。既定URLは `http://localhost:1234/v1` です。
3. OpenAI互換APIの場合は「カスタム」を選び、HTTPSのベースURL、モデルID、必要ならAPIキーを入力します。
4. 「接続設定を保存」後に「接続テスト」を実行します。
5. 「作戦」→「COA・検査」で生成方式を「外部LLM」に切り替えると3 COAを生成できます。採用案を選択後、「外部LLMで検証」でレッドチーム結果を取得できます。

APIキーはページ内メモリだけに保持し、プロジェクトJSON、JANUS JSON、履歴、AIログには保存しません。新規作成・JSON読込・ページ終了時に破棄されます。平文HTTPは `localhost`、`127.0.0.1`、`::1` だけを許可し、外部ホストにはHTTPSを要求します。リクエスト本文は現在選択中の作戦、割当済み部隊、その集計装備、作戦図の要約に限定されます。LM Studio側では `file://` からの接続を許可するCORS設定が必要です。

## CSVインポート

UTF-8のCSVを読み込めます。ヘッダーは次の形式です。

```csv
name,category,specs,notes
主力戦車,戦車,120mm滑腔砲,
HIFV-40,装甲車両,重装甲歩兵戦闘車,
```

## データ仕様

集計値はJSONに保存せず、読み込み時と編集時に部隊ツリーから導出します。

```text
Agg(n) = Direct(n) + sum(Agg(child))
```

旅団以上を示す `echelon.brigadeOrAbove` が `true` の部隊は直接割当UIを持ちません。独自階梯を追加しても集計ロジックは変わりません。

## 要件対応

初版では FR-101〜106、FR-201〜203、FR-301〜305、FR-401〜404、FR-501〜505、FR-601〜605、FR-701〜706、FR-801、FR-803〜804、FR-901〜903 を実装対象としています。

FR-606（ユーザー定義グリフ）は、独自兵科の登録まで対応し、独自SVGグリフ登録は次版対象です。
