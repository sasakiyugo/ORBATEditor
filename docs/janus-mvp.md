# JANUS フェーズ1 MVP 実装メモ

対象要件: 「旧世界統合作戦計画支援システム JANUS 要件定義書」版0.1（2026-07-19）

## 実装範囲

| 機能 | 実装内容 |
|---|---|
| F-01 | 1000×600座標系の作戦状況図。目標、経路、区域、フェーズラインを保存 |
| F-02 | 作戦名、期間、主体、敵、指揮官意図、最終状態、目的、失敗条件、ROE、制約等の構造化入力 |
| F-03 | ORBAT部隊のID参照割当、任務区分、配下人員・兵器ロールアップ |
| F-04 | 直接型・形成型・間接型の3 COA、採用案選択、構想本文修正 |
| F-06 | 必須項目、期間、距離÷速度、利用可能時間、部隊・兵器参照、重複・親子割当、作戦図・COAの検査 |
| F-08 | 五段落命令、付属書、JANUS JSON、印刷/PDF、提出ロック |
| F-09 | 2計画の並列表示、期間・地域・同一部隊の衝突候補表示 |
| F-10 | 提出時のORBAT・カタログ・定義・計画書本文の固定保存 |

## 意図的な制約

- 外部LLMは未接続である。COA生成は `localCoaGenerator` に分離した決定論的テンプレートであり、外部へのデータ送信はない。
- 世界地図画像・地理座標データが未統合のため、状況図は抽象座標系である。
- 兵器カタログに速度、航続距離、射程、輸送力、弾薬、燃料の構造化列がないため、兵器個別の到達性・兵站検査は警告として保留する。
- 三層情報モデル、認証、レッドチームAI、9軸評価、タイムライン衝突抽出、裁定結果のORBAT反映はフェーズ2以降の対象である。

## データ保持

作戦はプロジェクトJSONの `operations` 配列へ保存する。提出時は `snapshot` に参照部隊・兵器・定義を保存し、`frozenOrder` に五段落命令本文を固定する。提出済み計画を編集したい場合は「複製」で下書きを作成する。
