# ORBAT Editor

Current version: **0.5.1**

部隊階層、装備カタログ、自動集計、NATO APP-6風の兵科記号付き組織図を一元管理する、ローカル完結型ORBATエディタです。

## 使い方

1. リポジトリをダウンロードまたはZIPで取得します。
2. `index.html` を Chrome / Edge で開きます。
3. 編集後、ヘッダーの「JSON保存」でプロジェクトを保存します。

以前のZIPを展開済みの場合は、フォルダへの上書きではなく旧フォルダを削除してから最新版を展開してください。アプリUIは常時ダーク、組織図キャンバスと画像出力は白背景です。

依存関係、ビルド、サーバ、アカウントは不要です。外部へのデータ送信も行いません。

## 初版の主な機能

- 複数ルート対応の部隊ツリー（追加、削除、改名、並べ替え、隷属変更、配下ごと複製、検索、折りたたみ）
- 編集可能なエシュロン・兵科定義
- 兵器カタログのCRUD、CSVインポート、使用箇所表示、参照中削除の防止
- `兵器価格一覧(1).csv`から価格を除外した兵器1,651品目（名称・原種別・生産国）を初期収録
- 137の細分類を16の大分類へ整理し、元の分類は「原種別」として保持（収録兵器の「その他」は0件）
- 大規模カタログ向けの名称・区分・生産国検索と300件単位表示
- 旅団未満への装備・人員の直接割当
- 兵器割当時の区分フィルター（区分別の登録件数を表示）
- 旅団以上の全子孫ロールアップと区分別小計
- 所属フレーム、兵科、エシュロンを組み合わせたSVG兵科記号
- APP-6D準拠の所属フレーム形状と縦横比（味方3:2、敵・中立・不明は正方形境界）
- 常時ダークモード（ライトモードは廃止）
- 部分組織図、表示深度、人員・主要装備表示、ズーム・パン、エシュロン記号と重ならない接続線
- 組織図のSVG / PNG出力
- 単一JSONの保存・読込、集計CSV / Markdown出力、Undo / Redo、未保存離脱警告
- 循環参照の防止、階梯逆転と不正な直接割当の警告

初期プロジェクトには受け入れ試験用の師団・旅団・3個大隊が入っています。旅団の初期集計は、主力戦車88両、HIFV-40 40両、人員1,700名です。

初期カタログは旧世界固有兵器HIFV-40を含めて全1,652件です。固有兵器を指さない旧サンプル「主力戦車」「小銃」は削除し、サンプル戦車大隊にはM1A2を割り当てています。旧版JSONを読み込んだ場合は、旧「主力戦車」の数量をM1A2へ引き継いで移行します。価格情報は保存しません。

## CSVインポート

UTF-8のCSVを読み込めます。ヘッダーは次の形式です。

```csv
name,category,specs,notes
主力戦車,戦車,120mm滑腔砲,
HIFV-40,装甲車両,重装甲歩兵戦闘車,
```

## データ仕様

集計値はJSONに保存せず、読み込み時と編集時に部隊ツリーから導出します。

```text
Agg(n) = Direct(n) + sum(Agg(child))
```

旅団以上を示す `echelon.brigadeOrAbove` が `true` の部隊は直接割当UIを持ちません。独自階梯を追加しても集計ロジックは変わりません。

## 要件対応

初版では FR-101〜106、FR-201〜203、FR-301〜305、FR-401〜404、FR-501〜505、FR-601〜605、FR-701〜706、FR-801、FR-803〜804、FR-901〜903 を実装対象としています。

FR-606（ユーザー定義グリフ）は、独自兵科の登録まで対応し、独自SVGグリフ登録は次版対象です。

## スモークテスト

開発時のみ依存関係を導入し、次を実行できます。アプリの利用自体に `npm` は不要です。

```bash
npm install
npm test
```
